# STEEL FRONT — Mobile 3D FPS

Three.js 製のシングルファイル 3D FPS ゲーム(バトルフィールド風)。
PC・スマートフォンの両方に対応しています。

**現行モード: CONQUEST(10 vs 10 拠点占領戦)**

## 遊び方

`index.html` をブラウザで開くだけでプレイできます(ビルド不要)。

### ルール(コンクエスト)
- マップ上の 5 つの拠点 **A〜E** を占領して戦う 10 vs 10(プレイヤー + 味方AI 9 vs 敵AI 10)
- 拠点の円内に立つと占領ゲージが進行(人数が多いほど速い)
- **過半数(3 つ以上)の拠点を保持すると敵チームのチケットが減少**
- 兵士が戦死するとそのチームのチケットが -1
- 相手のチケットを 0 にしたチームの勝利
- 戦死しても 5 秒後に保持拠点 or HQ からリスポーン

### PC 操作
| 操作 | キー |
|---|---|
| 移動 | W / A / S / D |
| 視点 | マウス(画面クリックでポインターロック) |
| 射撃 | 左クリック |
| ダッシュ | Shift(前進中) |
| ジャンプ | Space |
| リロード | R |
| 乗り物に乗る / 降りる | E |

### スマートフォン操作
- **左画面ドラッグ**: 移動(前いっぱいに倒すとダッシュ)
- **右画面ドラッグ**: 視点操作
- **FIRE / JUMP / RELOAD** ボタン
- 乗り物に近づくと「**乗る**」ボタンが出現(ジープ・戦車に搭乗可能)

## マップ(v0.2)

300m × 300m のハイトフィールド地形。丘・高台・外周の山による高低差があり、
兵士・車両・弾道・視線(LoS)すべてが地形の影響を受けます。

```
        北
  A(丘の砦)          D(倉庫地区)   [赤HQ]
        \               /  \        /
         \             /    \      /
  西      C(中央市街地)      東
         /             \    /
        /               \  /
  B(村)                 E(高地基地)
  [青HQ]
        南
```

| 拠点 | 地区 | 特徴 |
|---|---|---|
| A | 北西の丘の砦 | 高台・砦壁・遮蔽が多い |
| B | 南西の村 | 三角屋根の民家・井戸・柵 |
| C | 中央市街地 | 高層建物・コンテナ・土のう |
| D | 北東の倉庫地区 | 倉庫・2段積みコンテナヤード |
| E | 南東の高地基地 | 高台・ヘリパッド・アンテナ塔 |

その他: 監視塔 ×2、岩場、樹木 40 本以上、道路網(拠点間を接続・地形追従)、遠景の山並み

## 主な機能
- **コンクエストモード**: 5 拠点の占領 + チケット制 + リスポーン
- **20 人の兵士**: 味方AI 9 体(青)+ 敵AI 10 体(赤)が拠点へ移動・交戦・占領を自律的に行う
- 乗り物: M151 ジープ(重機関銃)/ T-70 戦車(主砲)— 地形の傾斜に追従、坂で減速
- レーダー(拠点 A〜E・敵味方・車両を表示)/ チケット表示 / 占領ゲージ / キルフィード / ヒットマーカー
- WebAudio による効果音(外部アセット不要)
- WebGL コンテキストロスト自動復旧 / エフェクト完全プール化による負荷対策

## バージョン履歴

### v0.2(2026-07-11)
**ゲームモード刷新: 10 vs 10 コンクエスト(拠点占領制)**
- 5 つの占領拠点(A〜E)+ 両軍 HQ を追加。拠点円内の人数差で占領が進行
- チケット制: 各チーム 150 チケット。過半数拠点保持で敵チケットが毎秒減少、戦死でも -1。0 になったチームの敗北
- 10 vs 10: プレイヤー + 味方AI 9 体(青チーム)vs 敵AI 10 体(赤チーム)
  - AI は拠点の選定(距離・所有状況・味方の分散を考慮)→ 移動 → 交戦 → 占領を自律的に行う
  - AI 同士も交戦し、戦死した AI は HQ からリスポーン
  - 遠距離移動時はダッシュで移動時間を短縮
- プレイヤーのリスポーン: 戦死後 5 秒で保持拠点 or HQ から再出撃
- HUD 刷新: チケット表示、拠点状態ストリップ(A〜E)、占領進行バー、勝利/敗北画面

**マップ刷新: より広くリアルな地形**
- 300m × 300m(v0.1 の約 2.4 倍の面積)
- ハイトフィールド地形: 丘 7 箇所・外周の山・細かい起伏。移動・車両・弾道・視線すべてが高低差の影響を受ける
- 地区ごとの特色: 丘の砦 / 村(民家・井戸)/ 中央市街地 / 倉庫地区 / 高地基地(ヘリパッド)
- 拠点間を結ぶ道路網(地形追従・頂点カラーで描画)
- 頂点カラーによる地表の描き分け(草地 / 土 / 岩 / 道路)
- 岩場・樹木・監視塔・ドラム缶・木箱などの配置強化
- 車両は地形の傾斜に応じて姿勢が変化し、上り坂で減速

### v0.1.1(2026-07-11)
バグ修正・改善:
- **移動方向の修正**: 横を向いた状態で移動すると前後(上下入力)が反転して逆方向に移動してしまうバグを修正
  - カメラ空間 → ワールド座標への回転変換の符号誤りを修正(`updatePlayer` 内の `wx` / `wz` 計算)
- **戦車の砲塔の向きを修正**: 砲塔がエイム方向と 180° 逆を向いていた問題を修正(余分な `+Math.PI` を除去)
  - ジープの機銃塔も同じ計算を共有しているため、あわせて正しい向きになります
- **AI(CPU)の動きを改善**:
  - 障害物回避: 進行方向が塞がれている場合、左右にスライドして壁沿いに回り込むように
  - スタック検知: 一定時間ほぼ移動できていない敵は自動的に回避行動へ移行
  - 視線(LoS)に基づく行動: プレイヤーが見えない間は距離を保たず接近・回り込みを行う
  - エイム時間の導入: 視線が通った直後に即撃ちせず、約 0.35 秒のエイム時間を挟むことで理不尽さを緩和
  - 視線判定を間引き(約 0.25 秒間隔)して処理負荷を軽減
